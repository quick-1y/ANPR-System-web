## Соглашения по коду

### Именование функций и методов

В проекте используется Python-конвенция по разделению публичного и внутреннего API:

- **Публичные функции/методы** — имя в `snake_case` **без** ведущего `_`.
  - Примеры: `process_frame()`, `build_components()`.
- **Приватные (внутренние) функции/методы** — имя в `_snake_case` с ведущим `_`.
  - Примеры: `_run_channel()`, `_resolve_zone_eligibility()`.

Важно: это соглашение об уровне доступа по договорённости в команде (convention), а не жёсткое ограничение языка Python.
