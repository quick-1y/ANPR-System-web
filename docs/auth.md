# Аутентификация и управление пользователями

## JWT и аутентификация

Да, в проекте **по-прежнему используется JWT**.

- Выдача токена: `POST /api/auth/login`.
- Проверка токена: на защищённых API через `Authorization: Bearer <token>`.
- Для SSE/MJPEG поддерживается `?token=<jwt>` в query-параметре.
- Срок жизни JWT по умолчанию: 8 часов (`JWT_EXPIRATION_MINUTES=480`).

Публичные endpoint'ы без JWT:
- `GET /api/health`
- `POST /api/auth/login`

Все остальные endpoint'ы требуют валидный JWT.

## Модель доступа (актуальная)

В проекте используется простая модель на текущем этапе:
- доступ к разделам управляется tab-permissions;
- `superadmin` имеет полный доступ;
- debug-функции доступны только `superadmin`.

### Доступные tab-permissions

| Ключ | Название |
|---|---|
| `tab:obs` | Наблюдение |
| `tab:journal` | Журнал |
| `tab:zones` | Зоны |
| `tab:clients` | Клиенты |
| `tab:settings` | Настройки (все, кроме отладки) |

### Ограничения по ролям

- `admin`: можно назначать `tab:obs`, `tab:journal`, `tab:zones`, `tab:clients`, `tab:settings`.
- `operator`: можно назначать `tab:obs`, `tab:journal`, `tab:zones`, `tab:clients`.
- `superadmin`: технический root-аккаунт, bypass проверок permissions и единственный доступ к debug.

## Сводка по API-доступу

- Публично: `/api/health`, `POST /api/auth/login`.
- Авторизованный пользователь: базовые пользовательские API по своим tab-доступам.
- Требует `tab:settings`: глобальные настройки, контроллеры, backup/restore, retention, управление пользователями, список доступных permissions.
- Только `superadmin`: `/api/debug/*`.

## Роли и назначение прав

- `admin`: можно включать вкладки `Наблюдение`, `Журнал`, `Зоны`, `Клиенты`, `Настройки`.
- `operator`: можно включать вкладки `Наблюдение`, `Журнал`, `Зоны`, `Клиенты`.
- `superadmin`: технический аккаунт с полным доступом + единственный доступ к отладке.

## Auth API (кратко)

| Метод | URL | Доступ | Описание |
|---|---|---|---|
| `POST` | `/api/auth/login` | Публичный | Вход по логину/паролю, выдача JWT |
| `POST` | `/api/auth/logout` | Авторизованный | Аудит-лог выхода, токен удаляет клиент |
| `GET` | `/api/auth/me` | Авторизованный | Текущий пользователь |
| `GET` | `/api/permissions/available` | `tab:settings` | Список доступных tab-прав |

## Безопасность входа

- Включён rate-limit на вход: не более 5 неудачных попыток в минуту с одного IP (`HTTP 429`).
- Аккаунт `superadmin` создаётся автоматически на пустой базе (по текущей логике проекта).
